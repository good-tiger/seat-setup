# 강의 폴더 일괄 이미지→MD 변환 에이전트

부모 폴더 하나를 입력으로 받아, 그 안의 모든 대상 하위 폴더를 **병렬로** image-to-md 변환한다.

## 입력
- 부모 폴더 경로 한 개 (예: `C:\SSAFY\수업자료`)
- (선택) 출력 파일명 패턴이나 제외할 폴더 이름

## 절차

### 1단계: 대상 하위 폴더 수집
Glob/Bash로 부모 폴더 바로 아래의 디렉토리 목록을 가져온다.

**대상 판별 기준 (둘 중 하나라도 만족):**
- 하위 폴더 안에 `png/` 디렉토리가 있고 그 안에 이미지가 1개 이상
- 하위 폴더 자체에 이미지 파일(`.png .jpg .jpeg .webp .gif`)이 1개 이상

대상이 0개면 사용자에게 폴더 구조를 다시 확인하라고 알리고 종료한다.

### 2단계: 사용자에게 계획 보고
대상 폴더 목록과 각 폴더의 이미지 개수를 표/목록으로 보여준다. (한 번에 너무 많이 처리하기 전에 확인 의도)

### 3단계: 병렬 dispatch
대상 폴더 각각에 대해 `Agent` 도구로 sub-agent를 호출한다.

**중요 규칙:**
- **반드시 한 번의 메시지에 여러 Agent 호출을 동시에 넣어 병렬 실행한다.** (순차 호출 금지)
- sub-agent 설정:
  - `subagent_type`: `general-purpose`
  - `model`: `haiku`
  - `description`: `Convert {폴더명} images to MD`
- 각 sub-agent 프롬프트에 넣을 내용:
  - 대상 폴더 절대 경로
  - "image-to-md 스킬의 절차를 따라 이 폴더의 이미지를 마크다운으로 변환하라"
  - 출력 파일 경로: `{대상_폴더}/{대상_폴더_이름}.md`
  - 결과를 한 줄로 보고하라는 지시: `[Done] <파일경로> (이미지 N개)` 또는 `[Fail] <이유>`

### 4단계: 결과 취합
모든 sub-agent 결과를 모아서 표로 정리해 사용자에게 보고한다.

| 폴더 | 결과 | 이미지 수 | 출력 파일 |
|------|------|-----------|-----------|
| ... | Done/Fail | N | path.md |

실패한 폴더가 있으면 이유를 함께 표기한다. (성공한 결과는 보존)

## 모델
이 dispatcher 에이전트와 dispatch된 sub-agent **모두 Haiku를 사용한다.** 이미지 텍스트 추출/구조화 작업은 Haiku로 충분하며, 병렬 처리로 빠르게 끝낸다.

## 엣지 케이스
- **부모 폴더에 하위 폴더가 없음**: 단일 폴더 변환이라면 image-to-md 스킬을 직접 쓰라고 안내.
- **하위 폴더가 너무 많음 (20개 이상)**: 사용자에게 한꺼번에 진행할지 확인.
- **일부 sub-agent 실패**: 다른 결과는 그대로 보존하고, 실패한 폴더만 마지막에 모아 보고. 재시도는 사용자가 요청할 때만.
- **이미 같은 이름의 .md가 존재**: sub-agent에게 덮어쓰도록 지시 (기본 동작), 사용자가 명시적으로 보존을 요청한 경우만 건너뛴다.
