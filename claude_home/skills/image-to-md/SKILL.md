# Image to Markdown 스킬

이미지 폴더 안의 수업 교재/강의자료 이미지를 순서대로 읽어 텍스트를 **있는 그대로** 추출하고, 구조화된 마크다운 파일로 저장한다.

---

## 핵심 원칙

- **내용 보존 우선**: 텍스트를 임의로 생략하거나 요약하지 않는다. 이미지에 있는 모든 텍스트를 추출한다.
- **구조는 분석**: 텍스트의 순서와 계층(제목, 소제목, 본문, 목록, 표 등)은 이미지를 보고 판단해서 마크다운 문법으로 표현한다.
- **내용은 원문 그대로**: 단어, 문장, 수식, 코드 등을 바꾸거나 다듬지 않는다.

---

## 실행 절차

### 1단계: 입력 확인

사용자가 별도로 경로를 지정하지 않으면 기본 폴더는 `png/`이다.

```
기본 이미지 폴더: png/
기본 출력 파일명: {png폴더의 상위 폴더}.md  (예: parent/png → parent.md, parent/slides → parent.md)
```

사용자가 다른 경로나 출력 파일명을 지정한 경우 그것을 우선한다.

### 2단계: 이미지 목록 수집 및 정렬

`bash_tool`로 폴더 내 이미지 파일 목록을 가져온다.

```bash
ls {이미지_폴더}/ | sort
```

- 정렬 기준: **파일명 알파벳/숫자 오름차순** (sort 기본값)
- 지원 확장자: `.png`, `.jpg`, `.jpeg`, `.webp`, `.gif`
- 파일 목록을 사용자에게 먼저 보여주고 순서가 맞는지 확인한다 (선택사항, 파일이 10개 이상이면 권장)

### 3단계: 이미지별 텍스트 추출

각 이미지를 `view` 도구로 열어 시각적으로 분석한다.

**추출 규칙:**

| 이미지 요소               | 마크다운 변환 방식                                     |
| ------------------------- | ------------------------------------------------------ |
| 슬라이드/페이지 제목      | `## 이미지 N: {제목}` 또는 `## {제목}`                 |
| 소제목 / 섹션 헤더        | `###`, `####` 등 계층에 맞게                           |
| 본문 단락                 | 일반 텍스트 단락                                       |
| 글머리 기호 목록          | `- 항목`                                               |
| 번호 목록                 | `1. 항목`                                              |
| 표/그리드                 | 마크다운 표 문법                                       |
| 코드 블록                 | ` ```언어\n코드\n``` `                                 |
| 수식                      | LaTeX 인라인 `$수식$` 또는 블록 `$$수식$$`             |
| 강조(굵게/기울임)         | `**굵게**`, `*기울임*`                                 |
| 이미지 속 다이어그램/그림 | `[그림: 간략한 설명]` 으로 표기 (텍스트 레이블은 포함) |

**주의:**

- 텍스트가 두 열 이상으로 나뉜 경우, 자연스러운 읽기 순서(위→아래, 왼쪽→오른쪽)로 배치한다.
- 페이지 번호, 날짜, 저작권 표시 등 부가 정보도 삭제하지 않고 포함한다. (페이지 하단 등 적절한 위치에)
- 텍스트가 없는 순수 그림/사진 슬라이드는 `[이미지만 있는 슬라이드]`로 표기한다.

### 4단계: MD 파일 작성

모든 이미지 처리가 끝나면 `create_file` 도구로 마크다운 파일을 생성한다.

**파일 구조 템플릿:**

```markdown
# {폴더명 또는 사용자 지정 제목}

> 출처: {이미지\_폴더}/ | 총 {N}개 이미지 | 생성일: {날짜}

---

## 이미지 1: {제목 또는 파일명}

{추출된 텍스트}

---

## 이미지 2: {제목 또는 파일명}

{추출된 텍스트}

---

...
```

- 각 이미지 섹션은 `---` 구분선으로 나눈다.
- 이미지에 명시적인 제목이 있으면 그것을 섹션 제목으로 쓴다. 없으면 파일명을 사용한다.

### 5단계: 파일 저장 및 전달

```
저장 경로: {작업_디렉토리}/{출력_파일명}.md
```

`present_files` 도구로 사용자에게 파일을 전달한다.

---

## 엣지 케이스

- **이미지 폴더가 없을 때**: 사용자에게 경로를 다시 확인해달라고 요청한다.
- **이미지가 0개일 때**: 폴더는 있지만 지원 이미지 파일이 없다고 알린다.
- **텍스트가 매우 많은 이미지**: 잘라내지 말고 전부 추출한다. 길어도 괜찮다.
- **손으로 쓴 글씨(필기)**: 최대한 판독해서 추출하되, 불명확한 부분은 `[판독 불가]`로 표기한다.
- **혼합 언어(한/영 등)**: 원문 언어 그대로 유지한다. 번역하지 않는다.
